### 1. Requêtes de manipulation des données (Data Manipulation Language - DML)

Les requêtes de manipulation des données (DML) sont utilisées pour interagir directement avec les informations contenues dans tes tables. Elles permettent d'insérer, de mettre à jour ou de supprimer des lignes de données.

> [!IMPORTANT]
> **Règle d'or** : Les commandes DML agissent exclusivement sur le contenu des tables (les lignes) et non sur leur structure physique.

#### INSERT : Ajouter des données

La commande `INSERT` ajoute de nouveaux enregistrements dans une table.

```sql
INSERT INTO nom_table 
   (colonne1, colonne2, ...) 
VALUES 
   (valeur1, valeur2, ...);

```

> [!WARNING]
> **Points de vigilance clés** :
> * Tu dois scrupuleusement respecter l'ordre des colonnes définies dans la liste des variables.
> * Les valeurs textuelles ou temporelles doivent obligatoirement être encadrées par des guillemets simples ou doubles.
> * Les valeurs numériques s'écrivent sans aucun guillemet.
> * Les dates suivent par défaut le format standard `'YYYY-MM-DD'`.
> 
> 

#### UPDATE : Modifier des données existantes

La commande `UPDATE` modifie les informations déjà présentes au sein d'une table.

```sql
UPDATE nom_table
SET colonne1 = valeur1, colonne2 = valeur2, ...
WHERE condition;

```

> [!CAUTION]
> **Danger immédiat** : L'oubli de la clause `WHERE` lors d'un `UPDATE` est une erreur critique ! Sans condition restrictive, la modification s'appliquera instantanément à **TOUS** les enregistrements de la table.

#### DELETE : Supprimer des données

La commande `DELETE` retire définitivement un ou plusieurs enregistrements d'une table.

```sql
DELETE FROM nom_table
WHERE condition;

```

> [!CAUTION]
> Tout comme pour l'instruction `UPDATE`, l'omission de la clause `WHERE` videra **intégralement** le contenu de la table spécifiée. Cette action est irréversible en production sans sauvegarde préalable.

### 3. Requêtes d’interrogation (Data Query Language - DQL)

Les requêtes d'interrogation (DQL) forment le socle fondamental du travail d'analyse. Elles te permettent de formuler des questions précises pour trier les informations métiers exploitables.

#### SELECT : Extraire tes attributs

La commande `SELECT` extrait les données d'une table spécifiée.

```sql
SELECT 
   colonne1, colonne2 
FROM 
   nom_table;

```

> [!TIP]
> **Bonne pratique de production** : Bien que l'astérisque (`SELECT *`) soit pratique lors de tes phases d'exploration rapide, il est recommandé de spécifier explicitement le nom de tes colonnes en production afin d'optimiser l'espace en mémoire et les temps de réponse du SGBD.

#### L'alias (`AS`) : Renommer pour clarifier

Les alias modifient temporairement l'intitulé d'une colonne dans l'affichage du tableau final pour rendre le résultat compréhensible.

```sql
SELECT 
   firstname AS prenom, lastname AS nom
FROM 
   customer;

```

#### WHERE : Appliquer des conditions de filtrage

La clause `WHERE` filtre tes données pour ne retenir que les lignes répondant à des critères précis.

```sql
SELECT * FROM customer WHERE lastname = 'Dover';

```

##### Aperçu des principaux opérateurs logiques et relationnels :

* **Opérateurs relationnels** : `=`,`!=` (ou `<>`), `>`, `<`, `>=`, `<=`.
* **`LIKE` (Recherche textuelle dynamique)** : Utilisé avec `%` (zéro ou plusieurs caractères manquants) ou `_` (exactement un caractère manquant).
* **`IN` (Appartenance)** : Filtre par rapport à un ensemble fini de valeurs connues.
* **`BETWEEN` (Intervalle)** : Restreint les résultats entre une borne minimale et maximale incluses.
* **`IS NULL` / `IS NOT NULL**` : Isole les valeurs absentes ou présentes d'une colonne.
* **Combinateurs logiques** : `AND`, `OR`, `NOT` (Pense à structurer tes priorités de filtres à l'aide de parenthèses).

```sql
-- Isole les produits dont le prix est en dehors de la tranche [50 - 1000] avec du stock disponible
SELECT * FROM product WHERE (price < 50 OR price > 1000) AND stock > 10;

```

#### ORDER BY : Structurer l'ordre des lignes

La clause `ORDER BY` trie le jeu de données selon une ou plusieurs colonnes de manière ascendante (`ASC`, par défaut) ou descendante (`DESC`).

```sql
SELECT * FROM customer ORDER BY lastname ASC, firstname DESC;

```

#### LIMIT et OFFSET : Pagination des données

`LIMIT` restreint le volume maximal de lignes retournées, tandis qu'`OFFSET` définit la position de départ (saut de lignes). Idéal pour mettre en place des systèmes de pagination légers.

```sql
-- Récupère 10 clients en sautant les 5 premiers (affiche les lignes de 6 à 15)
SELECT * FROM customer LIMIT 10 OFFSET 5;

```

- a. Rien, la requête renverra une erreur de sécurité.
- b. Tous les enregistrements de la table seront définitivement supprimés.
- c. Seule la première ligne insérée sera retirée.

##### 2. Quelle clause permet de trier les lignes par ordre alphabétique ou numérique inverse ?
- a. SORT BY
- b. GROUP BY
- c. ORDER BY ... DESC

- a. LIMIT 11, 20
- b. LIMIT 20 OFFSET 11
